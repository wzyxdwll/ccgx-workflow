# Migration Guide: CCG v4.2 → v4.3

**Released**: 2026-05-04
**Phase scope**: P25 (pipeline-check) → P30 (release wrap)

> v4.3 is the **dynamic defense mechanisms** release. It does **not** introduce new user-facing features — instead it converts every v4.2.x release-blocker root cause (4.2.1 review-driven patch / 4.2.2 missing tarball file / 4.2.3 wrong plugin subagent_type) into automated infrastructure: pipeline-check / ground-truth-sampler / interface-auditor / fixtures auto-gen / commit-msg-review hook.

---

## TL;DR

- **No breaking changes** for end users on the command palette
- **Small startup overhead**: `/ccg:autonomous` Step 4.0 runs the ground-truth sampler (~50ms)
- **Small runtime overhead**: `triple`/`debate` verify wave gains a 3rd `interface-auditor` spawn (`fast` unchanged)
- **New opt-in git hook**: `~/.claude/hooks/ccg-commit-msg-review.cjs` installed but NOT auto-registered. Activate per-repo via three methods (symlink / Husky / `core.hooksPath`)
- **Phase-runner prompt tightened**: must `Read .context/ground-truth/latest.json` before referencing any plugin subagent_type / skill name / agent name (the v4.2 root cause)
- **5-step cold-start validation checklist** below — recommended after upgrade
- **Known race for v4.4**: parallel `phase-runner` commit attribution, recommend worktree isolation

## What's new

### 1. Pipeline-check helper (P25, commit `6378a6e`)

Defends against the v4.2.2 incident where `templates/commands/debate.md` was created by v4.1 P17 but never added to `package.json`'s `files` whitelist, breaking `/ccg:debate` across three releases.

**How it works**: helper runs `pnpm pack` (or `--dry-run`) and audits the resulting tarball file list against `templates/commands/*.md` glob. Any new template not whitelisted produces a clear error.

**Action required**: none for end users. Maintainers should run pipeline-check before release.

### 2. Ground-truth-sampler (P26, commit `fbf7c3c`)

Defends against the v4.2.0-2.2 incident where the system assumed `codex:codex-rescue` was the plugin subagent_type but the real Anthropic plugin marketplace uses `codex:rescue`.

**How it works**: `/ccg:autonomous` Step 4.0 (new) invokes the sampler skill, which inspects the actual installed plugins / skills / agents on disk and writes `.context/ground-truth/latest.json` with anonymized lists. The phase-runner prompt is **hard-constrained** to `Read` this file before quoting any plugin subagent_type, skill name, or agent name in spawn calls.

**Default behaviour change**: every `/ccg:autonomous` run now incurs ~50ms sampler overhead. If you have a tight CI loop and don't need the protection, you can short-circuit by writing a stale `latest.json` manually — but this defeats the purpose.

### 3. Interface-auditor specialist (P27, commit `af31f68`)

Defends against cross-phase interface debt (e.g. v4.2 P22 `buildVerifyWave` 95% duplicating P21 `planVerifyWave`).

**How it works**: `templates/commands/agents/interface-auditor.md` defines a specialist with a 5-item checklist:
1. SSoT violations (two helpers doing the same job)
2. Leftover code (unused helper after refactor)
3. Magic strings vs ground-truth (hardcoded `codex:codex-rescue` etc.)
4. Unvalidated assumptions about external interfaces
5. Public API drift between `src/index.ts` and consumers

`/ccg:autonomous` Step 4.4 verify wave spawns this specialist as a 3rd path **only on `--quality=triple|debate`** (fast skips it for wall-clock).

**Default behaviour change**: triple verify spawn count 2 → 3; debate verify spawn count 2 → 3 (in the final verify wave only). See updated `qualityTierE2E.test.ts` budget assertions.

### 4. Fixtures auto-generation (P28, commit `1602f0e`)

Defends against inline mocks drifting from real interfaces silently.

**How it works**: `scripts/regen-fixtures.ts` runs the sampler against your local install and writes `tests/fixtures/ground-truth/*.sample.json` (anonymized). Three test files (`challengerOrchestrator` / `debateOrchestrator` / `verifyOrchestrator`) load these fixtures alongside their original inline tests.

**Action required**: when you change a public schema (e.g. `ChallengerSummary`), run `pnpm regen-fixtures` to refresh the fixtures.

### 5. Commit-msg-review git hook (P29, commit `e6b6db0` + wire-in `89034a7`)

Defends against commit message ↔ diff inconsistency (e.g. `fix(p27)` but staged paths are all `phase-29/*`).

**How it works**: `templates/hooks/ccg-commit-msg-review.cjs` is a heuristic-only Node CJS script (no LLM, no network). 3 checks:
1. **Filename mention**: any file mentioned in message must be in `git diff --cached --name-only`
2. **Phase tag ↔ staged paths**: `feat(v4.3-p27)` must align with `phase-27` paths in diff
3. **Operation type ↔ diff**: `docs(...)` with no `.md` files in diff fails; `test(...)` with no test file fails

**Installation**: `ccg init` copies it to `~/.claude/hooks/ccg-commit-msg-review.cjs` but does **NOT** register it as a git hook (would clobber Husky / lefthook users).

**Activation (opt-in, per-repo)**:

```bash
# Option A: per-repo symlink (Unix)
cd /path/to/your/repo
ln -s ~/.claude/hooks/ccg-commit-msg-review.cjs .git/hooks/commit-msg
chmod +x .git/hooks/commit-msg

# Option B: Husky integration
echo "node ~/.claude/hooks/ccg-commit-msg-review.cjs \"\$1\"" >> .husky/commit-msg

# Option C: shared core.hooksPath
git config core.hooksPath ~/.claude/hooks/git
ln -s ~/.claude/hooks/ccg-commit-msg-review.cjs ~/.claude/hooks/git/commit-msg
```

See `~/.claude/hooks/README-commit-msg-review.md` for Windows notes (no symlinks without elevation; use copy or Husky).

**Bypass**: `git commit --no-verify` skips the hook. The hook also exits 0 (silent pass) on any infrastructure failure (git command non-zero, malformed message file) — never blocks legitimate commits due to its own bugs.

## Phase-runner prompt tightening

`templates/commands/agents/phase-runner.md` now requires:

> Before referencing **any** plugin subagent_type (e.g. `codex:rescue`), skill name, or registered agent name in your spawn calls or returned summaries, you **must** `Read .context/ground-truth/latest.json` and quote the exact name from `installed_plugins[*].subagent_type` / `installed_skills[*].name` / `installed_agents[*].name`. Do **not** fall back to training data or historical CHANGELOG / migration documents — they are **stale** by construction.

This is enforced at the prompt level; engine-level enforcement (rejecting spawn calls with unknown subagent_type) is left to Anthropic's plugin runtime.

## Validation checklist post-upgrade

This 5-step checklist exercises the new dynamic defense mechanisms. **Recommended after upgrade since CI dogfood cannot exercise real plugin spawn paths** (engine-level constraint, validated in v4.0.1 commit `a7cdffd`).

```bash
# Step 1. Install / refresh templates
ccg init --skip-prompt --skip-mcp --force
ls ~/.claude/hooks/
# Expect 4 files: ccg-context-monitor.js / ccg-statusline.js
#                  ccg-session-state.cjs / ccg-commit-msg-review.cjs
ls ~/.claude/agents/ccg/ | grep interface-auditor
# Expect: interface-auditor.md present

# Step 2. Run ground-truth-sampler manually (autonomous Step 4.0 does this for you)
node ~/.claude/skills/ccg/tools/ground-truth-sampler/scripts/sample.mjs
cat .context/ground-truth/latest.json | head -40
# Expect: installed_plugins / installed_skills / installed_agents sections,
#         valid JSON, anonymized, time-stamped

# Step 3. Pipeline-check (in this repo, not user repo)
pnpm pack --dry-run | tee /tmp/ccg-pack-audit.log
# Expect: every templates/commands/*.md file is present in the tarball list

# Step 4. Interface-auditor unit tests
pnpm test src/utils/__tests__/interfaceAuditor.test.ts
# Expect: 18 cases pass; covers all 5 finding types + lenient parser + severity helpers

# Step 5. Commit-msg-review hook adversarial test
mkdir /tmp/hook-test && cd /tmp/hook-test && git init
ln -s ~/.claude/hooks/ccg-commit-msg-review.cjs .git/hooks/commit-msg
chmod +x .git/hooks/commit-msg
echo "x" > a.txt && git add a.txt
git commit -m "fix(v4.3-p99): bogus phase tag for unrelated file"
# Expect: hook exits non-zero, stderr lists "phase-99 not in staged paths" or similar,
#         hint includes "--no-verify" escape note
git commit -m "chore: add a.txt"  # well-formed
# Expect: silent pass, exit 0
```

If any step fails, file at https://github.com/wangzy-ai/ccg-workflow/issues with the failing output.

## Known unverified items (latent bugs to watch)

1. **Real plugin spawn paths** — engine-level constraint (v4.0.1) prevents phase-runner from spawning `codex:rescue` / `gemini:rescue` plugin Agents. P25-P29 dogfood ran the orchestration logic via integration-test mocks. User cold-start is the first end-to-end exercise of plugin paths.

2. **commit-msg-review hook real activation** — CI cannot trigger git hooks meaningfully. Hook behavior is fully covered by 30 unit tests, but the first real per-repo activation (Step 5 above) is the first real-world signal.

3. **Sampler timing under sandbox** — sampler reads from `~/.claude/{plugins,skills,agents}/`. If you sandbox `/ccg:autonomous` heavily, ensure read access to those paths or sampler will return an empty manifest (which then trips phase-runner ground-truth assertions).

## Known race (deferred to v4.4)

v4.3 wave 1 dogfood exposed a **new race form** distinct from the v4.1 `src/index.ts` content-overwrite race:

- **Symptom**: parallel `phase-runner` instances each `git add <their files>`, then sequence into `git commit`. The committing phase-runner sweeps up files staged by sibling phases into its own commit, producing **commit message ↔ diff attribution mismatch** (content correct, attribution wrong).
- **Why P29 hook only partially helps**: heuristic #2 (phase tag ↔ staged paths) flags it when a wrong-phase commit is attempted, but doesn't prevent the underlying staging collision.
- **v4.4 recommended fix**: worktree-isolate each phase-runner. GSD's `code-fixer` (v4.0 P10 review-fix flow) already implements a 4-step transactional cleanup pattern (merge / remove / branch -D / rm sentinel) that maps directly to this use case.
- **Workaround until v4.4**: pass `--sequential` to `/ccg:autonomous` if you observe the race in your dogfood. This sacrifices wall-clock parallelism for attribution correctness.
